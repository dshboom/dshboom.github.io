C++学习笔记

C++学习笔记

第一节 —— C++概述

  • C++在C语言基础上添加了面向对象特性
  • C++是编译型的高级语言
  • C++充分相信程序员，你要对自己的行为负责哦！

第二节 —— C++环境安装

  • 安装Qt开发环境

第三节 —— 第一个C++程序

  • C++运行Hello world程序
#include <iostream>
// using namespace std 使用命名空间，听说不建议新手写
int main()
{
	std::cout << "Hello world!" << std::endl;
	return 0;
}

第四节 —— 基本数据类型

  • C++的几大主要类型：

    bool、int、float、double、char、void

  • unsigned修饰类型，如：unsigned int… …
  • sizeof(变量或类型)是一个关键字（不是函数），可以看变量或类型所占内存的字节数。
  • 想知道类型的最大范围可以使用limits库求得，示例程序：
#include <iostream>
#include <limits>
int main()
{
	unsigned long long a = std::numeric_limits<unsigned short int>::max();
	std::cout << a << std::endl;
	std::cin.get();
	return 0;
}
tip1：如果你测short类型的话，会发现是32767，而不是顺口的65535（2^16-1），因为有一个比特位拿去表示正负了，写成unsigned short即可。
tip2：a的最值要比被测值大哦，不然直接成了求a的最大值ψ(._. )>
  • 类型的别名：

C++中可以使用typedef关键字来指定类型别名，与原名等价，示例程序：

#include <iostream>
typedef unsigned long long ULL;
int main()
{
	ULL a = std::numeric_limits<unsigned short int>::max();
	std::cout << a << std::endl;
	std::cin.get();
	return 0;
}

第五节 —— 变量与作用域

  • 变量的生存期和作用域

    生存期：何时分配内存

    作用域：何地变量可见

  • 全局变量：静态生存期（一般来说程序结束消失）
  • 全局变量位置不能执行非定义语句，展开看示例：
    #include <iostream>
#include <limits>
int a,b;
int g_a;
g_a = a + b;// 编译错误！
int main()
{
	std::cin.get();
	return 0;
}
  • 注意：变量未初始化会出现一个随机的无意义数字
  • 局部变量：函数内部定义，函数执行时存在

    局部变量可以与全局变量同名

  • 变量赋值：深拷贝和浅拷贝

    浅拷贝：形式拷贝，无关语义

    深拷贝：A原来的内容覆盖掉，再也找不到

  • 大括号作用域
    • 大括号作用域可以嵌套
    • 局部作用域就是最外层大括号
    • 应尽量减少变量作用域

大括号限定变量的生存周期，示例程序：

#include <iostream>
int main()
{
	int a = 5,b = 8;
	{
		int t = a;
		a = b;
		b = t;
	}// 大括号限定t的生存周期
	std::cout << a << "," << b << std::endl;
	std::cout << t << std::endl;// 报错，未声明的标识符
	std::cin.get();
	return 0;
}

第六节 —— 常数和常量

  • 粗略的内存结构：只读区和数据区
    • 只读区：存放编译后的程序代码、常数图标、国际字符串
    • 数据区：由操作系统虚拟，工作栈、动态分配的内存变量、程序的工作空间… …
  • 常数（立即数、字面量）：
    • 字符常数：有些字符无法显示(如回车)，可以用ASCII码表示，也可以用转义字符（\n\t）。
    • 整型常数：默认int，长整数加LL

      例如：365 * 24 * 60 * 60 * 1000默认出整型，365 * 24 * 60 * 60 * 1000LL可出long long

    • 浮点数类型：f、d尾标表示
  • 常量：用const修饰的变量，占用数据区内存，编译器形式上保证你不能修改它
    • 常量必须初始化
    • 用处：
      • 一份拷贝、多处用到
      • 避免程序员笔误修改
      • 不易出现”魔法数字“
  • 枚举类型：把一组数据符号化，比如星期、日期等。

第七节 —— 修饰符

类型修饰符

  • 对大类型的细化，如short、long、signed、unsigned
  • const常量
  • volatile表示不要进行自动优化

储存修饰符

  • auto 是C++默认的类型，称为：自动变量、栈变量、局部变量（C++17废弃）
  • register——直接将变量放在寄存器中，速度快（C++17开始废弃）
  • static 修饰，静态空间，与全局变量同在
  • extern 跨文件的伪声明，告诉编译器全局变量会在另一个地方声明。

内存的类型

  • 程序分配的内存，主要在“栈”和“堆”里。
  • 只读代码：存代码、常数、定常资源
  • 静态空间：存放static变量、全局变量、常量
  • 栈区（stack）：自动变量，函数执行的上下文环境
  • 堆（heap）：程序运行中，动态分配

优美的栈 —— 后进先出的结构

  • 对比：堆是自由进出结构
  • 栈能实现沿着原路返回的功能
  • 特点：自动分配、自动释放
  • 缺点：栈溢出 StackOverflow

算数运算符

  • 整数、求余

    例如：将a%3可以把a限制在0~2的范围里

  • 浮点数运算有舍入误差
  • 字符型可以参与整型运算，示例程序（小写变大写）：
#include <iostream>
int main()
{
	char word = 'r';
	word -= 'a' - 'A';
	std::cout << word << std::endl;
	std::cin.get();
	return 0;
}

关系运算符

  • 大于、小于、等于、不等于… …
  • 一些陷阱
    • 不同类型的数之间的比较
    • 浮点数精确比较

逻辑运算符

  • 与或非的概念（&& || !）
  • 注意优先级
  • 短路求值特性，例如或运算，a||b，a如果是true，b表达式将被“短路”不运行。

其它

  • 赋值“=”也是运算符
  • “()”运算符是强制类型转换，示例程序：
#include <iostream>
int main()
{
	double a = 3.14;
	int b = (int)a;// 舍弃小数
	std::cout << b << std::endl;
	std::cin.get();
	return 0;
}
小寄巧：四舍五入这样写”a+=0.5”
  • 还有很多高级的，你现在整不明白👍

第九节 —— 选择与分支

  • 根据条件的真假，决定执行哪路代码
  • if的条件中尽量使用bool，其他类型会强转bool
  • 短路求值，和||的特性差不多
  • if(a||b)，a表达式为假时，b表达式不执行，示例程序：
#include <iostream>
int main()
{
	int a = 5;
	int b = 8;
	if(a > 10 && (b = 100))
	{
		std::cout << "这句话肯定不执行￣へ￣" << std::endl;
	}
	std::cout << b << std::endl;
	std::cin.get();
	return 0;
}
程序最终输出“8”，可见b=100被“短路”，未执行。
  • 尽量使用单if，避免else
  • 尽量避免if嵌套
  • 假设修正法写选择
    • 传统方法：
      bool leap_year = x % 4 == 0 && x % 100 != 0 || x % 400 == 0
    • 假设修正法：
      #include <iostream>
int main()
{
	int year = 2000;
	bool leap = false;// 假设不为润年
	if(year % 4 == 0) leap = true;
	if(year % 100 == 0) leap = false;
	if(year % 400 == 0) leap = true;// 不断修正得结果
	std::cin.get();
	return 0;
}
  • ?:表达式（唯一三目运算符）
  • 特点：是一个表达式，有返回值。
int d = a?b:c;
解释：看 a表达式是否成立，若成立，则返回 b，不成立，返回 c。

第十节 —— switch 语句

  • 分支过多时用
  • 和假设修正法有相似之处
switch(表达式){
case 1:
  代码块1;
  break;
case 2:
    代码块2;
    break;
default:
    默认处理
}
注意：switch 圆括号中的值必为：整数，字符 或者 枚举类型
  • 常见BUG：
    • 忘记了 default
    • 忘记了在每个case中写 break;

第十一节 —— while循环

  • 循环是程序逻辑的关键点，是代码复杂度的源泉，需要谨慎对待

最可靠的循环--死循环

while(true)
{
// 执行一些动作；
}

配合的出口语句：break if (条件) break;

示例： 对学生成绩进行分类，输出：A, B, C, D, E 5个等级

int score = 100;
int a = score - 60;
char level = 'E';
while(true){
if(a<0) break;
level--;// 能减的分数越多，等级越高
a -= 10; // a = a - 10;
if(level=='A') break; // 封顶
}
cout << level << endl;

示例： 已知正整数x，从低位到高位，输出每个十进制数位 提示：通过不断整除，消掉最后一位，直到变成0

#include <iostream>
int main()
{
	int a = 9000;
	while(true)
	{
		std::cout << a % 10 << std::endl;
		a /= 10;
		if(a == 0) break;
	}
	std::cin.get();
	return 0;
}

while 和 do...while

do-while 循环并不常用，所有情况，可以用内部的break来解决

第十二节 —— for循环

当我们已知执行次数的时候，用 for 循环更方便一些。

  • for 循环的语法：
for(初始化循环变量; 循环保持条件; 循环变量变化){
循环体
}

• 循环变量的生存期：循环变量仅仅在 for 语句范围内有效，是局部变量，放在栈中 循环变量的生存期，比循环体内的局部变量更长。
